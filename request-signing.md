# Request Signing

Every request to JustGold must be signed with HMAC-SHA256 using your `client_secret`.

## Signature inputs

Build the string to sign with these exact values:

```text
{HTTP_METHOD}
{REQUEST_PATH}
{TIMESTAMP}
{RAW_REQUEST_BODY}
```

Rules:

- `HTTP_METHOD` must be uppercase, for example `GET` or `POST`.
- `REQUEST_PATH` must contain only the path, for example `/v1/prices`.
- `TIMESTAMP` must match the value sent in `X-Timestamp`.
- `RAW_REQUEST_BODY` must be the exact JSON string sent over the wire.
- For requests without a body, use an empty string for `RAW_REQUEST_BODY`.

## Signature algorithm

1. Build the string to sign.
2. Compute `HMAC-SHA256(string_to_sign, client_secret)`.
3. Encode the digest as lowercase hexadecimal.
4. Send the result in `X-Signature`.

## Example

### Request

```http
POST /v1/buy/preview
X-Client-Id: jg_partner_123
X-Timestamp: 1767225600
Content-Type: application/json
```

```json
{"type":"buy","amount":5000,"currency":"AED"}
```

### String to sign

```text
POST
/v1/buy/preview
1767225600
{"type":"buy","amount":5000,"currency":"AED"}
```

### Node.js example

```js
import crypto from "node:crypto";

function createSignature({ method, path, timestamp, body, clientSecret }) {
  const rawBody = body ?? "";
  const stringToSign = [
    method.toUpperCase(),
    path,
    String(timestamp),
    rawBody,
  ].join("\n");

  return crypto
    .createHmac("sha256", clientSecret)
    .update(stringToSign, "utf8")
    .digest("hex");
}

const timestamp = Math.floor(Date.now() / 1000);
const body = JSON.stringify({
  type: "buy",
  amount: 5000,
  currency: "AED",
});

const signature = createSignature({
  method: "POST",
  path: "/v1/buy/preview",
  timestamp,
  body,
  clientSecret: process.env.JUSTGOLD_CLIENT_SECRET,
});

console.log({
  "X-Client-Id": process.env.JUSTGOLD_CLIENT_ID,
  "X-Timestamp": timestamp,
  "X-Signature": signature,
});
```

## Verification checklist

- Sign the exact JSON payload you send.
- Do not pretty-print a body after generating the signature.
- Generate a fresh timestamp for every request.
- Keep your server time synchronized with NTP.
- Treat signatures as invalid if either the path or body changes.

## Response signing

Every response returned by JustGold is also signed with HMAC-SHA256 using the same `client_secret`.

## Response headers

Verify these headers on every JustGold response:

| Header | Description |
| --- | --- |
| `X-Timestamp` | Timestamp generated by JustGold for the response. |
| `X-Signature` | HMAC-SHA256 signature generated by JustGold. |

## Response verification inputs

Build the response string to verify with these exact values:

```text
{HTTP_STATUS}
{TIMESTAMP}
{RAW_RESPONSE_BODY}
```

Rules:

- `HTTP_STATUS` is the numeric status code returned by JustGold, for example `200` or `201`.
- `TIMESTAMP` must match the value received in the response `X-Timestamp` header.
- `RAW_RESPONSE_BODY` must be the exact raw response body returned by JustGold.
- For empty responses, use an empty string for `RAW_RESPONSE_BODY`.

## Response verification algorithm

1. Build the response string to verify.
2. Compute `HMAC-SHA256(string_to_verify, client_secret)`.
3. Encode the digest as lowercase hexadecimal.
4. Compare it with the value received in `X-Signature`.
5. Process the response only if the signatures match.

## Response example

### Response

```http
HTTP/1.1 200 OK
X-Timestamp: 1767225602
X-Signature: <justgold_response_signature>
Content-Type: application/json
```

```json
{"buyPrice":382.45,"sellPrice":379.10,"currency":"AED"}
```

### String to verify

```text
200
1767225602
{"buyPrice":382.45,"sellPrice":379.10,"currency":"AED"}
```

### Node.js example

```js
import crypto from "node:crypto";

function verifyResponseSignature({
  status,
  timestamp,
  body,
  receivedSignature,
  clientSecret,
}) {
  const rawBody = body ?? "";
  const stringToVerify = [
    String(status),
    String(timestamp),
    rawBody,
  ].join("\n");

  const expectedSignature = crypto
    .createHmac("sha256", clientSecret)
    .update(stringToVerify, "utf8")
    .digest("hex");

  return crypto.timingSafeEqual(
    Buffer.from(expectedSignature, "utf8"),
    Buffer.from(receivedSignature, "utf8"),
  );
}
```

## Response verification checklist

- Verify the signature before parsing the response into downstream business logic.
- Use the exact raw response body returned by JustGold.
- Reject responses with missing `X-Timestamp` or `X-Signature` headers.
- Reject responses whose signatures do not match.
- Log verification failures without logging the `client_secret`.
