# Webhooks

JustGold sends webhook calls to your configured endpoint when important events happen in your integration.

Your endpoint must verify the webhook signature before processing the event.

## Supported events

- `transaction.status_changed`
- `customer.created`

## Webhook request

JustGold sends webhook events as `POST` requests with a JSON body.

```http
POST <your_webhook_url>
Content-Type: application/json
X-Timestamp: 1767225600
X-Signature: <webhook_signature>
```

## Headers

Verify these headers on every webhook request:

| Header | Required | Description |
| --- | --- | --- |
| `X-Timestamp` | Yes | Unix timestamp in seconds generated by JustGold. |
| `X-Signature` | Yes | HMAC-SHA256 signature generated by JustGold. |

## Payload format

Each webhook payload includes the event type and a flexible payload object.

```ts
export interface WebhookPayload {
  eventType: WebhookEventType;
  payload: {
    [key: string]: unknown;
  }
}
```

```json
{
  "eventType": "transaction.status_changed",
  "payload": {
    "transactionId": "txn_6818744f3f1b2c7a9d5e4321",
    "status": "Completed"
  }
}
```

## Signature verification

Webhook signatures are generated with HMAC-SHA256 using your `client_secret`.

Build the string to verify with these exact values:

```text
{TIMESTAMP}
{RAW_REQUEST_BODY}
```

Rules:

- `TIMESTAMP` must match the value received in the `X-Timestamp` header.
- `RAW_REQUEST_BODY` must be the exact raw request body received by your endpoint.
- Do not parse, reformat, or stringify the JSON before verifying the signature.
- Process the webhook only if the signature is valid.

## Verification algorithm

1. Read the raw request body.
2. Build the string to verify.
3. Compute `HMAC-SHA256(string_to_verify, client_secret)`.
4. Encode the digest as lowercase hexadecimal.
5. Compare it with the value received in `X-Signature`.
6. Process the event only if the signatures match.

## Node.js example

```js
import crypto from "node:crypto";

function verifyWebhookSignature({
  timestamp,
  body,
  receivedSignature,
  clientSecret,
}) {
  const rawBody = body ?? "";
  const stringToVerify = [
    String(timestamp),
    rawBody,
  ].join("\n");

  const expectedSignature = crypto
    .createHmac("sha256", clientSecret)
    .update(stringToVerify, "utf8")
    .digest("hex");

  if (expectedSignature.length !== receivedSignature.length) {
    return false;
  }

  return crypto.timingSafeEqual(
    Buffer.from(expectedSignature, "utf8"),
    Buffer.from(receivedSignature, "utf8"),
  );
}
```

## Verification checklist

- Reject webhook requests with missing `X-Timestamp` or `X-Signature` headers.
- Verify the signature before parsing the payload into business logic.
- Use the exact raw request body received by your server.
- Use a constant-time comparison for signatures.
- Reject timestamps outside your allowed clock-skew window.
- Log verification failures without logging the `client_secret`.

## Handling webhooks

- Return a `2xx` response only after your endpoint accepts the event.
- Use `eventType` to determine how to process the `payload`.
- Do not trust the payload until signature verification succeeds.
- Keep webhook processing fast. For longer work, store the event and process it asynchronously.
